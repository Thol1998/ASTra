  8  This script reads an input socket connected to the remote client and process an request when received. 
  9  Requests must be sent on the port 180007, responses are sent on the port 18008. 
 10   
 11  (0) Traffic lights COORDINATES request: COO tllId1 ... tllIdN 
 12          If no traffic light ID is specified, the geographic coordinates of every traffic light will be sent 
 13   
 14  (1) Traffic lights COORDINATES response: COO tllId1 lon1 lat1 ... tllIdN lonN latN (N in [0-X] if all traffic lights were requested (See constants for X value)) 
 15          If ALL TRAFFIC LIGHTS were requested, these messages are followed by an end message (response): TLL END 
 16   
 17  (2) Traffic lights DETAILS request: GET tmsLogin tllId zoom answerDetailsLevel 
 18          The zoom is an integer between 100 (min) and 100 000 (max). 100 means a 100% zoom, the entire map can fit in the screen. 
 19          answerDetailsLevel = 0 => (3) 
 20          answerDetailsLevel = 1 => (4) 
 21           
 22  (3) SHORT traffic lights DETAILS answer: DET tmsLogin screenshotPath currentPhaseIndex nextSwitchTime 
 23          The screenshotPath is an absolute path of a PNG file centered on a junction which has a traffic light 
 24          The current phase index is an integer between 0 and N 
 25          nextSwitchTime is a value in milliseconds. This one refers to the next traffic light state change 
 26          Note: if the operating system the server is running on is based on POSIX, the screenshotPath will be equal to null 
 27           
 28  (4) FULL traffic lights DETAILS answer: DET tmsLogin screenshotPath currentPhaseIndex nextSwitchTime state0 duration0 ... stateN durationN 
 29          See (3) 
 30          A phase is defined by a state and a duration. The phases order defines the traffic light behaviour 
 31              - State: string which is equals to the number of lanes the traffic light manage 
 32              - Duration: time in milliseconds for which a state will be displayed. The next state is then displayed 
 33           
 34  (5) SHORT SET traffic lights details request: SET tllId currentPhaseIndex 
 35          tllId: SUMO traffic light id 
 36          currentPhaseIndex: SUMO index corresponding to the current traffic light phase in the complete phases definition 
 37   
 38  (6) FULL SET traffic lights details request: SET tllId currentPhaseIndex state0 duration0 ... stateN durationN 
 39          For detailed parameters pieces of information, see (5) and (4) 
 40           
 41  (7) ACKNOWLEDGE or ERROR response after a SET((5) or (6)) request: ACK returnCode 
 42          OK = 0 
 43          Invalid phase index = 1 
 44          Invalid phase state = 2 
 45          Invalid phase duration = 3 
 46           
 47  (8) ERROR response when an invalid request is received : ERR 40 
 48   
 49  (9) Traffic lights dictionary: 
 50          - Key = SUMO edge ID 
 51          - Value = SUMO traffic light ID which is located at the end of the edge
